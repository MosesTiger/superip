### Software Requirements Specification (SRS)
## 1. Introduction

### 1.1 Purpose
이 문서는 영화 시나리오 작성 지원 도구를 개발하기 위한 요구사항을 정의합니다. 본 소프트웨어는 작가들이 생성형 AI를 사용하여 시나리오를 작성, 수정 및 평가할 수 있도록 돕는 것을 목표로 합니다. 주요 기능은 다음과 같습니다:
- 드라마 데이터 자료 확보
- 생성형 AI를 활용한 시나리오 작성 및 추천
- 흥행도 예측
- 실시간 피드백 제공
- UI/UX 개선을 통한 효율적인 시나리오 작성 지원

### 1.2 Scope
본 프로젝트는 작가들이 AI를 활용하여 시나리오를 작성하고 수정하는 과정을 지원하는 웹 애플리케이션을 개발하는 것입니다. 주요 사용자 그룹은 영화 시나리오 작가들입니다. 이 소프트웨어는 리액트(React)와 Next.js를 기반으로 개발될 것입니다.

### 1.3 Definitions, Acronyms, and Abbreviations
- **AI**: Artificial Intelligence (인공지능)
- **UI**: User Interface (사용자 인터페이스)
- **UX**: User Experience (사용자 경험)
- **SRS**: Software Requirements Specification (소프트웨어 요구사항 명세서)
- **TF-IDF**: Term Frequency-Inverse Document Frequency (용어 빈도-역문서 빈도)

### 1.4 References
- [영화산업에서 기계학습 활용의 현 단계: 흥행 예측과 추천 시스템을 중심으로]

### 1.5 Overview
이 문서는 프로젝트의 요구사항을 상세히 기술합니다. 시스템의 전반적인 설명, 사용자 인터페이스 요구사항, 하드웨어 및 소프트웨어 인터페이스, 성능 요구사항, 보안 요구사항 등을 다룹니다. 또한, 주요 시스템 기능에 대한 설명과 기술적 세부사항을 포함합니다.

## 2. Overall Description

### 2.1 Product Perspective
본 소프트웨어는 작가들이 AI를 통해 시나리오를 효율적으로 작성하고, 중간에 새로운 내용을 수집하며, 새로운 이야기를 창작할 수 있도록 돕는 웹 애플리케이션입니다. 시스템은 다음과 같은 주요 구성 요소로 이루어집니다:
- **프론트엔드**: 사용자 인터페이스를 제공하며, React와 Next.js로 구현됩니다.
- **백엔드**: 서버 로직과 데이터베이스 관리를 담당하며, Node.js와 Express로 구현됩니다.
- **데이터베이스**: 시나리오 데이터를 저장하기 위해 PostgreSQL을 사용합니다.
- **AI 모듈**: GPT 기반의 생성형 AI를 활용하여 시나리오 작성 및 추천을 지원합니다. OpenAI의 GPT API를 사용합니다.

### 2.2 Product Functions
- **드라마 데이터 자료 확보**: 영화 시나리오와 관련된 데이터를 수집하고 분석합니다.
- **생성형 AI를 활용한 시나리오 작성**: GPT를 통해 시나리오 작성 및 추천을 지원합니다.
- **흥행도 예측**: TF-IDF 알고리즘을 사용하여 대사의 독창성과 개연성을 기반으로 흥행도를 예측합니다.
- **실시간 피드백 제공**: 작성 중인 시나리오에 대해 실시간 피드백을 제공합니다.
- **UI/UX 개선**: 직관적이고 효율적인 사용자 인터페이스를 제공합니다.

### 2.3 User Characteristics
- **작가**: 시나리오를 작성하는 영화 작가들로, AI를 통해 시나리오 작성과 수정을 도와줍니다.
- **평가자**: 시나리오를 평가하고 피드백을 제공하는 전문가들입니다.

### 2.4 Constraints
- **보안**: 사용자 데이터와 시나리오 내용은 암호화되어야 합니다.
- **성능**: 시스템은 실시간 피드백을 제공할 수 있어야 하며, 고가용성을 유지해야 합니다.
- **호환성**: 다양한 브라우저와 호환되어야 합니다.

## 3. External Interface Requirements

### 3.1 User Interfaces
- **로그인 페이지**: 사용자 인증 및 권한 관리.
- **대시보드**: 프로젝트 상태, 시나리오 작성 진행 상황 및 통계 제공.
- **시나리오 편집기**: AI 추천 기능과 피드백을 제공하는 시나리오 작성 도구.
- **피드백 페이지**: 시나리오 평가와 피드백 제공.

### 3.2 Hardware Interfaces
- 웹 브라우저가 실행 가능한 모든 디바이스 (PC, 태블릿, 스마트폰)에서 접근 가능해야 합니다.

### 3.3 Software Interfaces
- **GPT API 연동**: OpenAI의 GPT API를 사용하여 시나리오 생성 및 추천.
- **데이터베이스 연동**: PostgreSQL을 사용하여 시나리오와 사용자 데이터를 저장 및 관리.
- **인증 시스템**: OAuth 2.0을 사용하여 사용자 인증 및 권한 관리.
- **메일 서비스 연동**: SMTP를 통해 이메일 알림 및 보고서 전송.

### 3.4 Communications Interfaces
- **HTTP/HTTPS**: 데이터 전송에 사용.
- **WebSocket**: 실시간 피드백 제공에 사용.

## 4. System Features

### 4.1 드라마 데이터 자료 확보
**기능 설명**: 영화 시나리오와 관련된 데이터를 수집하고 분석합니다.
- **입력**: 영화 제목, 줄거리, 등장인물 데이터.
- **출력**: 분석된 데이터, 영화 흥행도 예측.

#### Use Case: 드라마 데이터 수집 및 분석
1. **사용자**가 영화 제목, 줄거리, 등장인물 데이터를 입력합니다.
2. 시스템이 입력된 데이터를 **데이터베이스**에 저장합니다.
3. 시스템은 저장된 데이터를 분석하여 유의미한 정보를 추출합니다.
4. 분석 결과는 사용자에게 **대시보드**를 통해 제공됩니다.

### 4.2 생성형 AI를 활용한 시나리오 작성
**기능 설명**: GPT 기반의 AI를 활용하여 시나리오 작성 및 추천을 지원합니다.
- **입력**: 사용자가 입력한 시나리오 텍스트.
- **출력**: AI가 생성한 시나리오 텍스트, 추천 내용.

#### Use Case: 시나리오 작성 및 AI 추천
1. **사용자**가 시나리오 텍스트를 입력합니다.
2. 입력된 텍스트는 **백엔드 서버**를 통해 **데이터베이스**에 저장됩니다.
3. **백엔드 서버**는 저장된 텍스트를 **GPT API**에 전송합니다.
4. **GPT API**가 입력된 텍스트를 기반으로 시나리오 추천 및 생성 결과를 반환합니다.
5. 시스템은 반환된 결과를 **사용자 인터페이스**에 표시합니다.

### 4.3 흥행도 예측
**기능 설명**: TF-IDF 알고리즘을 사용하여 대사의 독창성과 개연성을 기반으로 흥행도를 예측합니다.
- **입력**: 시나리오 텍스트.
- **출력**: 흥행도 예측 점수.

#### Use Case: 흥행도 예측
1. **사용자**가 시나리오 텍스트를 입력합니다.
2. 입력된 텍스트는 **백엔드 서버**를 통해 **데이터베이스**에 저장됩니다.
3. 시스템은 저장된 텍스트에 대해 **TF-IDF 알고리즘**을 적용하여 흥행도를 예측합니다.
4. 예측 결과는 사용자에게 **대시보드**를 통해 제공됩니다.

### 4.4 실시간 피드백 제공
**기능 설명**: 작성 중인 시나리오에 대해 실시간 피드백을 제공합니다.
- **입력**: 사용자가 입력한 시나리오 텍스트.
- **출력**: 피드백 메시지, 수정 제안.

#### Use Case: 실시간 피드백 제공
1. **사용자**가 시나리오 텍스트를 입력합니다.
2. 입력된 텍스트는 **WebSocket**을 통해 **백엔드 서버**에 전송됩니다.
3. **백엔드 서버**는 입력된 텍스트를 실시간으로 분석하여 피드백을 생성합니다.
4. 생성된 피드백은 **사용자 인터페이스**에 실시간으로 표시됩니다.

### 4.5 UI/UX 개선
**기능 설명**: 직관적이고 효율적인 사용자 인터페이스를 제공합니다.
- **입력**: 사용자의 인터랙션.
- **출력**: 시각적 피드백, 사용성 향상.

#### Use Case: 사용자 인터페이스 개선
1. **사용자**가 시스템과 상호작용합니다.
2. **프론트엔드**는 사용자의 상호작용을 처리하고 시각적 피드백을 제공합니다.
3. **백엔드 서버**는 사용자 데이터를 분석하여 UI/UX 개선에 필요한 정보를 제공합니다.
4. 시스템은 지속적으로 UI/UX를 개선하여

 사용자 경험을 최적화합니다.

## 5. Non-functional Requirements

### 5.1 Performance Requirements
- **응답 시간**: 사용자가 요청을 보낸 후 응답을 받기까지의 시간은 평균 2초 이내여야 합니다.
- **처리량**: 시스템은 초당 최소 100개의 시나리오 문장을 처리할 수 있어야 합니다.
- **확장성**: 서버를 추가하여 처리 능력을 확장할 수 있어야 합니다.
- **자원 사용량**: 서버의 CPU 사용률은 70%를 초과하지 않아야 합니다.
- **고가용성**: 시스템의 가용성은 99.9% 이상을 유지해야 합니다.

### 5.2 Security Requirements
- **데이터 암호화**: 모든 데이터는 전송 중 및 저장 시 암호화되어야 합니다.
- **사용자 인증**: OAuth 2.0을 통한 사용자 인증 및 권한 관리.
- **보안 감사**: 정기적인 보안 감사 및 취약점 분석을 통해 시스템의 보안 상태를 점검해야 합니다.

### 5.3 Usability Requirements
- **직관적인 UI**: 사용자 인터페이스는 직관적이고 사용하기 쉬워야 합니다.
- **도움말 문서**: 사용자 매뉴얼과 도움말 문서를 제공하여 사용자가 쉽게 학습할 수 있도록 지원해야 합니다.
- **접근성**: 접근성 표준을 준수하여 모든 사용자가 접근할 수 있어야 합니다.

### 5.4 Maintainability Requirements
- **유지보수성**: 시스템은 변경 사항이 발생할 때 쉽게 수정될 수 있어야 합니다.
- **모듈화**: 모듈화된 아키텍처를 채택하여 각 모듈이 독립적으로 유지보수될 수 있도록 해야 합니다.
- **문서화**: 소스 코드는 잘 구조화되어 있으며, 주석과 문서화가 충분히 이루어져야 합니다.

### 5.5 Compatibility Requirements
- **브라우저 호환성**: 시스템은 다양한 운영 체제와 브라우저에서 호환되어야 합니다.
- **표준 준수**: API와 데이터 형식은 표준을 준수하여 다른 시스템과의 상호 운용성을 보장해야 합니다.

## 6. Other Requirements

### 6.1 Reporting Requirements
- 시스템은 작성된 시나리오의 상태와 흥행도 예측 결과를 주기적으로 보고할 수 있어야 합니다.

### 6.2 Audit Requirements
- 시스템은 모든 중요한 작업과 변경 사항에 대한 로그를 기록하고 이를 감사할 수 있어야 합니다.

### 6.3 Training Requirements
- 시스템 사용자를 위한 교육 자료와 세션을 제공해야 합니다.
